## What it does

The **Financial Data Collector** pulls a vessel's budget allocation and year-to-date actual spend by category, plus underlying transaction detail, and files a timestamped snapshot. It pairs with the [Financial Analyzer](/skills/financial-analyzer): one fetches, one interprets.

## When to use it

- "Get budget data for POSUN"
- "Check OPEX numbers"
- "Pull financial data"
- "Refresh budget figures"

## What it captures

- Budget allocation by category (technical, crew, lube, victualling, port disbursements, repairs, etc.)
- Year-to-date actual spend per category
- Transaction list within each category (date, vendor, amount, description)
- Burn-rate metrics (monthly average, last 3 months)
- Currency and exchange-rate context

## How it works

1. Looks up the vessel by IMO.
2. Pulls the latest budget and spend records from the configured finance source.
3. Writes a timestamped evidence record under the vessel's financial case folder.
4. Updates the case so any reviewing agent can see new evidence is available.

## What it produces

- Structured budget and spend data consumed by the analyzer
- Plain-English summary covering total budget, total spend, headline variance
- Filing metadata (timestamp, source, sequence number)

Every collection is append-only — historical financial state is preserved.

## Related skills

- **financial-analyzer** — senior skill that turns this snapshot into a budget review
- **purchase-data-collector** — sister collector for procurement detail
- **status-rollup** — fleet-wide aggregation
