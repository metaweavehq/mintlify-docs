## What it does

The **Crew Data Collector** captures the current crew roster on a vessel — every officer and rating with rank, contract dates, certificate validity, nationality, and join date — and files a timestamped snapshot. It pairs with the [Crew Analyzer](/skills/crew-analyzer): one fetches, one interprets.

## When to use it

- "Get crew info for POSUN"
- "Who's onboard?"
- "Crew check"
- "Pull manning data"

## What it captures

- Full roster: rank, name, nationality, sign-on date, contract end
- Certificate validity per officer (CoC, GMDSS, STCW endorsements, medicals)
- Minimum safe manning configuration for the vessel
- Recent rest-hour records where the source provides them
- Active reliefs in planning

## How it works

1. Looks up the vessel by IMO.
2. Pulls the current crew roster and certificate inventory from the configured source.
3. Writes a timestamped evidence record under the vessel's crew case folder.
4. Updates the case so any reviewing agent can see new evidence is available.

## What it produces

- Structured crew data consumed by the analyzer
- Plain-English summary covering total crew, rank coverage, expiry counts
- Filing metadata (timestamp, source, sequence number)

Every collection is append-only — historical crew composition is preserved.

## Related skills

- **crew-analyzer** — senior skill that turns this snapshot into a manning review
- **status-rollup** — fleet-wide aggregation
