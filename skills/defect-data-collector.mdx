## What it does

The **Defect Data Collector** pulls the open and recently closed defect list for a vessel — equipment, severity, age, status, blocker — and files a timestamped snapshot. It pairs with the [Defect Analyzer](/skills/defect-analyzer): one fetches, one interprets.

## When to use it

- "Any defects?"
- "Equipment issues"
- "Check defects for AQUILA"
- "Pull defect register"

## What it captures

- Every open defect with: equipment / system, description, severity, opened date, status
- Recently closed defects with closure date and resolution
- Blocker codes where defects are stuck (parts, port call, vendor, class approval)
- Class-related conditions and PSC-linked items

## How it works

1. Looks up the vessel by IMO.
2. Pulls the current defect register from the configured source.
3. Writes a timestamped evidence record under the vessel's defect tracking case folder.
4. Updates the case so any reviewing agent can see new evidence is available.

## What it produces

- Structured defect data consumed by the analyzer
- Plain-English summary covering open count, severity distribution, recently closed
- Filing metadata (timestamp, source, sequence number)

Every collection is append-only — historical defect state is preserved.

## Related skills

- **defect-analyzer** — senior skill that turns this register into a defect review
- **pms-data-collector** — sister collector for planned maintenance
- **status-rollup** — fleet-wide aggregation
