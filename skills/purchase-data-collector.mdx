## What it does

The **Purchase Data Collector** captures the current procurement state for a vessel — open POs, approval status, supplier, ETAs, budget by category — and files a timestamped snapshot. It pairs with the [Purchase Analyzer](/skills/purchase-analyzer): one fetches, one interprets.

## When to use it

- "PO data"
- "Procurement status"
- "Purchase check"
- "PO check"

## What it captures

- Every open PO with: number, requestor, items, value, supplier, status, ETA
- Approval history and lead-time at each stage
- Supplier per-PO and aggregate performance metrics
- Budget allocation and actual spend per category
- Safety-critical flag where source provides it

## How it works

1. Looks up the vessel by IMO.
2. Pulls open POs, supplier info, and budget data from the configured source.
3. Writes a timestamped evidence record under the vessel's purchase case folder.
4. Updates the case so any reviewing agent can see new evidence is available.

## What it produces

- Structured procurement data consumed by the analyzer
- Plain-English summary covering total open value, in-transit count, headline blockers
- Filing metadata (timestamp, source, sequence number)

## Related skills

- **purchase-analyzer** — senior skill that turns this snapshot into a procurement review
- **financial-data-collector** — sister collector for budget detail
- **status-rollup** — fleet-wide aggregation
