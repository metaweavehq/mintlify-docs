## What it does

The **Voyage Data Collector** captures everything operationally relevant to a vessel's current voyage — leg-by-leg distances, speeds, consumption, weather, and port calls — and files a timestamped snapshot. It pairs with the [Voyage Analyzer](/skills/voyage-analyzer): one fetches, one interprets.

## When to use it

- "Pull voyage data"
- "Get noon reports for AQUILA"
- "Voyage check"
- "Voyage scan"

## What it captures

- Voyage legs with start / end position, distance, average speed
- Daily noon-report data: position, speed, FOC by grade, weather
- Port calls with arrival, departure, port stays, consumption in port
- Weather observations (wind, sea, current) per leg
- Charter Party reference where the source links it

## How it works

1. Looks up the vessel by IMO.
2. Pulls the latest voyage data from the configured source.
3. Writes a timestamped evidence record under the vessel's voyage case folder.
4. Updates the case so any reviewing agent can see new evidence is available.

## What it produces

- Structured voyage data consumed by the analyzer
- Plain-English summary covering current voyage, legs, ETA, headline performance
- Filing metadata (timestamp, source, sequence number)
