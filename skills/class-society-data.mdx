## What it does

The **Class Society Data Collector** pulls a vessel's complete class portfolio directly from the class society's portal — ABS today, with the same pattern extending to other societies. It uses browser automation to walk the portal pages a surveyor would visit, capturing both the structured data each page loads and a screenshot of the page itself.

The collector pairs with the [Class Society Analyzer](/skills/class-society-analyzer): one fetches, one interprets.

## When to use it

- "Fetch class data for AQUILA"
- "Get ABS data"
- "Run class collection"
- "Scrape the class portal"

## What it captures

| Page | Data |
|------|------|
| Vessels list | Fleet roster as the class society sees it |
| Overview | Class status, register details, key dates |
| Certificates | Statutory and class certificates with expiries |
| Surveys | Due dates, completion status, credit windows |
| Findings | Open and closed findings, severity, age |
| Assets | Equipment condition flags, recommendations |

A timestamped screenshot of each page is captured alongside the structured data so any reviewer can verify the recorded numbers came from the live portal.

## How it works

1. Reads the configured class portal URL and credentials.
2. Opens the portal in a visible browser so a human supervisor can watch the run if desired.
3. Runs a headless scraper in parallel that walks the portal pages, capturing each page's API response and a full-page screenshot.
4. Bundles structured data + screenshots into a single timestamped evidence record under the vessel's class case folder.

## What it produces

- **Structured class data** — surveys, certificates, findings, assets — consumed by the analyzer
- **Page screenshots** — visual evidence for every portal page visited
- **Plain-English summary** — vessel, class status, headline counts

Every collection is append-only — re-running never overwrites earlier snapshots, so historical class state is preserved.

## Why visuals matter

Class disputes hinge on what the portal showed at a specific date. Pairing structured data with timestamped screenshots gives Technical Superintendents and auditors a defensible record — the numbers and the source page that produced them.
